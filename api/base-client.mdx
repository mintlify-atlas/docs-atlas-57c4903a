## Overview

`BaseClient` provides core functionality used by all GLAM SDK clients. It manages connections, programs, transactions, and common utilities.

## Constructor

```typescript
new BaseClient(config?: GlamClientConfig)
```

<ParamField path="config" type="GlamClientConfig" optional>
  Configuration options for the client
  
  <Expandable title="properties">
    <ParamField path="provider" type="anchor.Provider" optional>
      Anchor provider instance. If not provided, uses `anchor.AnchorProvider.env()`
    </ParamField>
    <ParamField path="wallet" type="Wallet" optional>
      Wallet instance for signing transactions
    </ParamField>
    <ParamField path="cluster" type="ClusterNetwork" optional>
      Solana cluster network. Auto-detected from provider URL if not specified
    </ParamField>
    <ParamField path="statePda" type="PublicKey" optional>
      State account public key to connect to a vault
    </ParamField>
    <ParamField path="jupiterApiKey" type="string" optional>
      Jupiter API key for swap operations
    </ParamField>
    <ParamField path="jupiterApiClient" type="JupiterApiClient" optional>
      Custom Jupiter API client instance
    </ParamField>
    <ParamField path="useStaging" type="boolean" optional>
      Use staging programs instead of production
    </ParamField>
  </Expandable>
</ParamField>

## Transaction methods

### intoVersionedTransaction

```typescript
async intoVersionedTransaction(
  tx: Transaction,
  options: TxOptions
): Promise<VersionedTransaction>
```

Converts a legacy transaction into a versioned transaction with compute budget optimization and lookup tables.

<ParamField path="tx" type="Transaction" required>
  Legacy transaction to convert
</ParamField>

<ParamField path="options" type="TxOptions" required>
  <Expandable title="properties">
    <ParamField path="signer" type="PublicKey" optional>
      Transaction signer. Defaults to `client.signer`
    </ParamField>
    <ParamField path="computeUnitLimit" type="number" optional>
      Compute unit limit. Auto-calculated via simulation if not provided
    </ParamField>
    <ParamField path="getPriorityFeeMicroLamports" type="(tx: VersionedTransaction) => Promise<number>" optional>
      Function to calculate priority fee
    </ParamField>
    <ParamField path="maxFeeLamports" type="number" optional>
      Maximum fee in lamports
    </ParamField>
    <ParamField path="useMaxFee" type="boolean" optional>
      Use maximum fee limit
    </ParamField>
    <ParamField path="preInstructions" type="TransactionInstruction[]" optional>
      Instructions to prepend
    </ParamField>
    <ParamField path="postInstructions" type="TransactionInstruction[]" optional>
      Instructions to append
    </ParamField>
    <ParamField path="lookupTables" type="PublicKey[] | AddressLookupTableAccount[]" optional>
      Address lookup tables
    </ParamField>
    <ParamField path="simulate" type="boolean" optional>
      Throw error if simulation fails
    </ParamField>
  </Expandable>
</ParamField>

<Note>
This method automatically:
- Simulates the transaction to compute optimal CU limit
- Adds compute budget instructions
- Includes GLAM-specific lookup tables
- Fetches fresh blockhash
</Note>

### sendAndConfirm

```typescript
async sendAndConfirm(
  tx: VersionedTransaction | Transaction,
  additionalSigners?: Keypair[]
): Promise<TransactionSignature>
```

Signs, sends, and confirms a transaction.

<ParamField path="tx" type="VersionedTransaction | Transaction" required>
  Transaction to send
</ParamField>

<ParamField path="additionalSigners" type="Keypair[]" optional>
  Additional keypair signers
</ParamField>

Returns the transaction signature.

```typescript
const tx = await client.intoVersionedTransaction(transaction, {});
const signature = await client.sendAndConfirm(tx);
console.log("Transaction:", signature);
```

<Note>
This method:
- Uses a dedicated TX RPC endpoint on mainnet if configured via `TX_RPC` env var
- Skips preflight since simulation already occurred
- Waits for confirmation via WebSocket or polling
- Throws `GlamError` with parsed program logs on failure
</Note>

### fetchGlamStates

```typescript
async fetchGlamStates(filterOptions?: {
  owner?: PublicKey;
  delegate?: PublicKey;
  type?: string;
}): Promise<StateModel[]>
```

Fetches all GLAM vaults with optional filters.

<ParamField path="filterOptions" type="object" optional>
  <Expandable title="properties">
    <ParamField path="owner" type="PublicKey" optional>
      Filter by vault owner
    </ParamField>
    <ParamField path="delegate" type="PublicKey" optional>
      Filter by delegate
    </ParamField>
    <ParamField path="type" type="string" optional>
      Filter by account type (e.g., "tokenizedVault")
    </ParamField>
  </Expandable>
</ParamField>

```typescript

  type: "tokenizedVault"
```
